## 1. 현재 시스템 현황

### 1-1. 기술 스택

```
현재 아키텍처 (로컬 환경)
┌─────────────────────────────────┐
│    프론트엔드 (브라우저)          │
├─────────────────────────────────┤
│  - HTML/CSS/JavaScript           │
│  - 모듈화 구조 (8개 폴더)         │
│  - XLSX.js (엑셀 처리)            │
│  - 글래스모피즘 UI                │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│   IndexedDB (브라우저 로컬)       │
├─────────────────────────────────┤
│  - companies (거래처)             │
│  - reports (보고서)               │
│  - employees (직원)               │
│  - changeHistory (변경이력)       │
└─────────────────────────────────┘
```

### 1-2. 현재 문제점

| 문제 | 설명 | 영향 |
|------|------|------|
| **데이터 격리** | 각 사용자 브라우저에만 저장 | 팀 협업 불가 |
| **데이터 손실 위험** | 캐시 삭제 시 전체 손실 | 업무 중단 위험 |
| **백업 부재** | 중앙 백업 시스템 없음 | 재해 복구 불가 |
| **확장 불가** | 다중 접속 불가능 | 30명 목표 달성 불가 |
| **동기화 불가** | 데이터 공유 불가능 | 실시간 협업 불가 |

## 2. 배포 필요성

### 2-1. 목표 아키텍처

```
배포 후 아키텍처
┌────────────────────────────────────────┐
│        사용자 (30명 동시 접속)           │
│  ┌──────┐  ┌──────┐  ┌──────┐         │
│  │영업담당│  │영업담당│  │ 관리자│  ...   │
│  └───┬──┘  └───┬──┘  └───┬──┘         │
└──────┼─────────┼─────────┼─────────────┘
       │         │         │
       └─────────┼─────────┘
                 ↓ HTTPS
       ┌─────────────────────────┐
       │    호스팅 서버            │
       ├─────────────────────────┤
       │  프론트엔드 (정적 파일)   │
       │         ↓                │
       │  백엔드 (Node.js)        │
       │         ↓                │
       │  MySQL 데이터베이스       │
       └─────────────────────────┘
```

### 2-2. 주요 변경사항

| 구분 | 현재 | 변경 후 |
|------|------|---------|
| **데이터베이스** | IndexedDB (브라우저) | MySQL (서버) |
| **백엔드** | 없음 | Node.js + Express |
| **통신 방식** | 로컬 함수 호출 | REST API (HTTPS) |
| **인증** | 로컬 저장 | JWT 토큰 |
| **데이터 공유** | 불가능 | 30명 실시간 공유 |
| **백업** | 수동 | 자동 일일 백업 |
